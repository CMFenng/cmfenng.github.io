Git 安装

到 Git 官网 获取安装包。

注意：从官方链接下载安装包可能需要翻墙！

安装过程建议保持默认设置，一路 next！

检查是否安装成功，能获取到对应版本号，则说明安装成功。

$ git --version

注册 GitHub

进入 GitHub 官网，已经有 GitHub 账号的可以直接 登录，还没有的需要 注册，根据提示输入用户名、Email、密码即可完成注册。

注意：GitHub 账号用户名一旦注册成功就不能修改。比如我的用户名是 mingfengst

Git 配置

配置 SSH

在终端设置 user.name 和 user.email

$ git config --global user.name "your github user.name"
$ git config --global user.email "your github user.email"

生成 ssh 密钥

$ ssh-keygen -t rsa -C "your github user.email"

此时，在用户文件夹下就会有一个新的文件夹.ssh，里面有刚刚创建的 ssh 密钥文件 id_rsa 和 id_rsa.pub。

添加 SSH 到 GitHub

用户头像 → Settings → SSH and GPG keys → New SSH key → 将 id_rsa.pub 中的内容复制到 Key 文本框中，然后点击 Add SSH key（添加 SSH）按钮。

Git 常用命令

# 查看 git 的版本号
$ git --version
# 查看 git 的安装路径（Mac）
$ which git
# 查看 git 的安装路径（Win）
$ where git
# 克隆仓库，默认与远程主机的版本库同名。如果要指定不同的目录名，可以将目录名作为第二个参数
$ git clone <版本库地址> <本地目录名>
# 查看分支，* 为当前分支
$ git branch
# 新建分支
$ git branch <分支名>
# 切换分支
$ git checkout <分支名>
# 新建并切换到新分支
$ git checkout -b <分支名>
# 删除分支（两者有区别）
$ git branch -d <branch_name>
$ git branch -D <branch_name>
# 恢复被删除的分支
$ git branch <branch_name> <hash_val>
# 显示整个仓库的 commit，包括所有分支的 commit，甚至包括已经撤销的 commit
$ git reflog
# 
$ git add .
# 
$ git commit -m '提交备注'
# 
$ git push origin <分支名>
# 
$ git update-ref -d HEAD

删除 git 版本库中的 commit

如果在代码中写了一些测试密码之类比较隐私的东西，而且已经 commit 到 git 中，那么就需要删除 commit 记录。

$ git reset HEAD~1
# 加上参数 --hard 则不保留当前更改
$ git reset --hard HEAD~1
$ git push --force

$ git revert HEAD
# 还是可以查到被删除的 commit 记录
$ git reflog

合并两个 git 仓库

有 2 个 git 仓库：A 和 B，想要把这两个仓库进行合并，并且要保留以前的 commit 记录，也就是说要把 A 仓库中的文件移动到 B 仓库中，并且 A 仓库之前的 commit 记录也要在 B 仓库中保留。

A 仓库的目录树：

B 仓库的目录树：

第一步：将 A 作为远程仓库，添加到 B 仓库中，设置别名为 B_othername。

[r2] $ git remote add <r2_othername> <r2_path>
# eg:
# [r2] $ git remote add other ../r1/

第二步：从 A 仓库中抓取数据到 B 仓库。

[r2] $ git fetch <r2_othername>
# eg:
# [r2] $ git fetch other
# From ../r1/
#  * [new branch]      master     -> other/master

第三步：将 A 仓库中抓取的 master 分支作为新分支 checkout 到 B 仓库，新分支名设置为 A_branch。

[B] $ git checkout -b <A_branch> <B_othername>/master
# eg:
# [B] $ git checkout -b A_branch other/master
# Branch 'A_branch' set up to track remote branch 'master' from 'other'.
# Switched to a new branch 'A_branch'

第四步：切换回 B 仓库的 master 分支。

[r2] $ git checkout master
# eg:
# Switched to branch 'master'
# Your branch is up to date with 'origin/master'.

第五步：将 A_branch 分支合并入 B 仓库的 master 分支。

[r2] $ git merge <A_branch> --allow-unrelated-histories
# eg:
# [r2] $ git merge A_branch --allow-unrelated-histories
# Merge made by the 'recursive' strategy.
