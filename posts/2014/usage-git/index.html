2014-06-18
  • 工具

Git 安装

到 Git 官网 获取安装包。

注意：从官方链接下载安装包可能需要翻墙！

安装过程建议保持默认设置，一路 next！

检查是否安装成功，能获取到对应版本号，则说明安装成功。

$ git --version

注册 GitHub

进入 GitHub 官网，已经有 GitHub 账号的可以直接 登录，还没有的需要 注册，根据提示输入用户名、Email、密码即可完成注册。

注意：GitHub 账号用户名一旦注册成功就不能修改。比如我的用户名是 mingfengst。

Git 配置

配置 SSH

在终端设置 user.name 和 user.email。

$ git config --global user.name "your_github_user.name"
$ git config --global user.email "your_github_user.email"

生成 ssh 密钥。

$ ssh-keygen -t rsa -C "your_github_user.email"

此时，在用户文件夹下就会有一个新的文件夹.ssh，里面有刚刚创建的 ssh 密钥文件 id_rsa 和 id_rsa.pub。

添加 SSH 到 GitHub

用户头像 → Settings → SSH and GPG keys → New SSH key → 将 id_rsa.pub 中的内容复制到 Key 文本框中，然后点击 Add SSH key（添加 SSH）按钮。

Git 常用命令

# 查看 git 的版本号
$ git --version
# 查看 git 的安装路径（Mac）
$ which git
# 查看 git 的安装路径（Win）
$ where git
# 克隆仓库，默认与远程主机的版本库同名。如果要指定不同的目录名，可以将目录名作为第二个参数
$ git clone <版本库地址> <本地目录名>
# 查看分支，* 为当前分支
$ git branch
# 新建分支
$ git branch <分支名>
# 切换分支
$ git checkout <分支名>
# 新建并切换到新分支
$ git checkout -b <分支名>
# 删除分支（两者有区别）
$ git branch -d <branch_name>
$ git branch -D <branch_name>
# 恢复被删除的分支
$ git branch <branch_name> <hash_val>
# 回退到指定版本，保留当前源码的更改
$ git reset <commit_id>
# 回退到指定版本，放弃当前源码的更改（添加了参数 --hard）
$ git reset --hard <commit_id>
# 回退到上 n 个版本
$ git reset HEAD~n
$ git reset --hard HEAD~n
# 使用上述方法进行回退后，推送的时候需要添加参数 --force
$ git push --force
# 版本回退，不会擦除“回退”之后的 commit_id，而是正常的当做一次 commit，产生一次新的操作记录，所以可以 push，不会让你再 pull
$ git revert <commit_id>
# 查看命令历史记录（可查看到被删除的 commit 记录）
$ git reflog
# 查看提交历史记录
$ git log
# 添加更改到缓冲区
$ git add .
# 提交
$ git commit -m '提交备注'
# 推送到远程库
$ git push origin <分支名>
# 
$ git update-ref -d HEAD

删除 git 版本库中的 commit

假设一开始 git 仓库本地和远程都是 a -> b -> c，想把 HEAD 回退到 b，那么在本地就变成了 a -> b，这时，如果没有远程库，那么接下来怎么操作都可以，比如 a -> b -> d，但是在有远程库的情况下，git push 会失败，因为远程库是 a -> b -> c，本地库是 a -> b -> d，以下是两种解决方案：

  1. git push 的时候用 --force 参数，强制把远程库变成 a -> b -> d，但是在大部分公司里是不允许这么做的。
  2. 使用 git revert <commit_id> 做一个反向操作，实现以退为进，把本地变成 a -> b -> c -> d，注意 b 和 d 文件快照内容一模一样，但是 commit_id 肯定不同，再 push 到远程也会变成 a -> b -> c -> d。

git revert 不同于 git reset，它不会擦除 “回退” 之后的 commit_id，而是正常的当做一次 commit，产生一次新的操作记录，所以可以 push，不会让你再 pull。

合并 git 版本库

有 2 个 git 仓库：A 和 B，想要把这两个仓库进行合并，并且要保留之前的 commit 记录，也就是说要把 A 仓库中的文件移动到 B 仓库中，并且 A 仓库之前的 commit 记录也要在 B 仓库中保留。

A 仓库的目录树：

A
├── .git
├── 01-demo.js
└── README.md

B 仓库的目录树：

B
├── .git
├── 02-demo.js
└── README.md

合并仓库后的目录树：

.
├── .git
├── 01-demo.js
├── 02-demo.js
└── README.md

第一步：进入 B 仓库的目录，将 A 作为远程仓库，添加到 B 仓库中，并设置别名为 other。

[B] $ git remote add other ../A    # ../A 为 A 仓库的路径

第二步：从 A 仓库中抓取数据到 B 仓库。

[B] $ git fetch other
# remote: Counting objects: 3, done.
# remote: Compressing objects: 100% (3/3), done.
# remote: Total 3 (delta 0), reused 0 (delta 0)
# Unpacking objects: 100% (3/3), done.
# From ../A
#  * [new branch]      master     -> other/master

第三步：将 A 仓库中抓取的 master 分支作为新分支 checkout 到 B 仓库，并设置新分支名为 A_branch。

[B] $ git checkout -b A_branch other/master
# Branch 'A_branch' set up to track remote branch 'master' from 'other'.
# Switched to a new branch 'A_branch'

第四步：完成上一步后，当前分支是 A_branch，切换回 B 仓库的 master 分支。

[B] $ git checkout master
# Switched to branch 'master'

第五步：将 A_branch 分支合并入 B 仓库的 master 分支。

[B] $ git merge A_branch
# fatal: refusing to merge unrelated histories

如果出现以上错误，在命令后添加 --allow-unrelated-histories 即可。

[B] $ git merge A_branch --allow-unrelated-histories
# 合并时出现冲突的情况：
# Auto-merging README.md
# CONFLICT (add/add): Merge conflict in README.md
# Automatic merge failed; fix conflicts and then commit the result.

# 合并成功的情况：
# Merge made by the 'recursive' strategy.

如果合并时发生冲突，那么需要先解决冲突后，再 commit 文件。

$ git add .
$ git commit -m 'Update'

之后 git log --pretty=oneline 可以看到两个合并后的仓库的 commit 也保留了下来。

拆分 git 版本库

现有一个名为 Case 的 git 仓库，里面包含了多个项目或者模块的文件夹，现在要把每个文件夹独立出来作为一个 git 仓库，并且保留之前各自提交的 commit 记录。

要拆分的 git 仓库目录树：

.
├── .git
├── other
├── vue-musicPlayer
└── weapp-QQmusicPlayer

以拆分 vue-musicPlayer 文件夹为例：

第一步：进入 Case 目录，将 Case 仓库中关于 vue-musicPlayer 的提交信息抽取出来作为新的分支。

[Case] $ git subtree split -P <folder_name> -b <branch_name>
# eg: git subtree split -P vue-musicPlayer -b vue-musicPlayer
# Created branch 'vue-musicPlayer'

第二步：新建 vue-musicPlayer 仓库，并从 Case 仓库中拉取 vue-musicPlayer 分支的内容，即可成为独立的仓库。

[Case] $ cd ..
[~] $ mkdir vue-musicPlayer
[~] $ cd vue-musicPlayer
[vue-musicPlayer] $ git init
[vue-musicPlayer] $ git pull ../Case vue-musicPlayer
# remote: Counting objects: 30, done.
# remote: Compressing objects: 100% (26/26), done.
# remote: Total 30 (delta 0), reused 27 (delta 0)
# Unpacking objects: 100% (30/30), done.
# From ../Case
#  * branch            vue-musicPlayer -> FETCH_HEAD
